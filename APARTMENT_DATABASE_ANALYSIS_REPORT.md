## アパート査定の課題

### 他の物件種別との比較

| 物件種別 | 変動係数 | 主要な価格決定要因 |
|----------|----------|-------------------|
| マンション | 約50-60% | 立地・築年数 |
| 戸建て | 約60-70% | 土地・建物の組み合わせ |
| 土地 | 約70-80% | 立地・面積 |
| **アパート** | **104.6%** | **収益性・構造・立地の複合要因** |

### アパート査定の困難な点

1. **価格変動が非常に大きい**（変動係数104.6%）
2. **収益性データが欠損**（想定利回り100%欠損）
3. **物件個別性が高い**（同一条件の類似物件が少ない）
4. **取引件数が少ない**（10,000件 vs 他物件種別の数十万件）

## 即時査定に必要な入力項目の提案

### 現行フォームの入力項目

- 物件種別
- 都道府県
- 市区町村
- 面積（建物面積）
- 築年数

### アパート選択時に追加すべき項目

#### 必須項目

| 項目 | 入力形式 | 理由 |
|------|----------|------|
| **建築構造** | 選択式（木造/軽量鉄骨/鉄骨造/RC/SRC） | 価格差が大きい（RC/SRCは木造の約1.5倍） |
| **階建** | 選択式（1階/2階/3階/4階以上） | 価格差が大きい（3階は2階の約1.8倍） |

#### 推奨項目

| 項目 | 入力形式 | 理由 |
|------|----------|------|
| 土地面積 | 数値入力（㎡） | 収益性評価に重要 |
| 総戸数 | 数値入力 | 収益性評価に重要 |

### フォーム設計案

```
物件種別: [土地] [戸建て] [マンション] [アパート]

【アパート選択時のみ表示】
├─ 建築構造: [木造▼] ← 必須
├─ 階建: [2階▼] ← 必須
├─ 土地面積: [___] ㎡ ← 任意
└─ 総戸数: [___] 戸 ← 任意
```

## 査定ロジックの提案

### アパート専用の査定アルゴリズム

1. **基本価格算出**: 同一都道府県・市区町村の類似物件から中央値を算出
2. **建築構造調整**: 木造を基準に、RC/SRCは+50%、鉄骨造は+20%
3. **階建調整**: 2階建てを基準に、3階以上は+80%
4. **築年数調整**: 築10年を基準に、築年数による減価
5. **面積調整**: 建物面積・土地面積による補正

### 信頼度の表示

アパートは価格変動が大きいため、以下の信頼度表示を推奨：

| 条件 | 信頼度 |
|------|--------|
| 類似物件50件以上 | 70-80% |
| 類似物件20-49件 | 50-70% |
| 類似物件10-19件 | 30-50% |
| 類似物件10件未満 | 30%以下 |

## 結論

アパートの即時査定を実現するには、**建築構造**と**階建**の2項目を追加入力項目として設けることが必須です。これにより、価格変動の大きなアパート物件でも、ある程度の精度で概算価格を提示できます。

ただし、アパートは収益物件としての性質上、想定利回りや賃料収入などの情報がないと正確な査定は困難です。即時査定では「参考価格」として提示し、詳細査定は面談で行う旨を明記することを推奨します。
