# Assets Directory

このディレクトリは、HYConsulting LPで使用するアセットを管理します。

## ディレクトリ構成

```
assets/
├── images/          # 画像ファイル
│   ├── logo/       # ロゴファイル
│   ├── hero/       # ヒーロー画像
│   ├── services/   # サービス紹介用画像
│   ├── features/   # 強み・特徴用画像
│   ├── clients/    # お客様の声用画像
│   └── icons/      # アイコン素材
└── documents/      # ドキュメントファイル
    ├── policies/   # プライバシーポリシーなど
    └── guides/     # ガイドドキュメント
```

## 画像ガイドライン

### ファイル形式
- **推奨形式**: WebP（STUDIO対応）
- **代替形式**: PNG（透過が必要な場合）、JPEG（写真）
- **SVG**: ロゴやアイコン

### 画像サイズ推奨値
- **ヒーローイメージ**: 1920x1080px
- **セクション背景**: 1920x600px
- **サービスアイコン**: 256x256px
- **お客様の声（プロフィール）**: 200x200px
- **OGP画像**: 1200x630px
- **ファビコン**: 512x512px

### ファイル命名規則
- 小文字とハイフンを使用
- 説明的な名前を付ける
- 例: `hero-main-image.webp`, `service-icon-consulting.png`

### 最適化
- STUDIOにアップロード前に画像を最適化
- ファイルサイズは可能な限り小さく（目安: 500KB以下）
- 品質と容量のバランスを考慮

## アップロード方法

### STUDIOへのアップロード
1. STUDIOエディタを開く
2. 左サイドバーの「アセット」タブをクリック
3. 「アップロード」ボタンをクリック
4. ファイルを選択またはドラッグ&ドロップ
5. アップロードされた画像を使用

### このリポジトリでの管理
- 元となる高解像度画像をこのディレクトリに保存
- バージョン管理のために重要な画像のみをコミット
- 大きなファイルは `.gitignore` に追加を検討

## 著作権・ライセンス

### 使用する画像のソース
- **自社撮影**: 権利確認不要
- **ストックフォト**: ライセンスを確認
  - 推奨サイト: Unsplash, Pexels, Pixabay
- **イラスト**: 商用利用可能か確認
  - 推奨サイト: unDraw, Storyset, Iconfinder

### 注意事項
- 著作権のある画像は許可を得てから使用
- ライセンス情報をドキュメント化
- クライアントから提供された画像は適切に管理

## 画像の更新フロー
1. 画像を作成または入手
2. 最適化（圧縮、リサイズ）
3. このディレクトリに保存
4. STUDIOにアップロード
5. エディタで配置
6. プレビューで確認
7. 変更をコミット
