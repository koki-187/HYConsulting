# データベース確認チェックリスト

## 目的
不動産検索システムのデータベースが実際に構築されているか、正確に確認するための手順書。
ユーザーへの報告前に必ず実施すること。

## 必須確認項目

### 1. データベーススキーマの確認
- [ ] `webdev_execute_sql` でテーブル一覧を取得
- [ ] `real_estate_aggregated` テーブルの存在確認
- [ ] テーブル構造の確認（カラム一覧）

### 2. 実データの件数確認
- [ ] 全レコード件数の確認: `SELECT COUNT(*) FROM real_estate_aggregated`
- [ ] 都道府県別件数の確認: `SELECT prefecture, COUNT(*) FROM real_estate_aggregated GROUP BY prefecture`
- [ ] 神奈川県のレコード件数確認
- [ ] 横浜市戸塚区のレコード件数確認

### 3. データ品質の確認
- [ ] サンプルデータの取得（最初の10件）
- [ ] 価格データの妥当性確認
- [ ] 取引件数の妥当性確認
- [ ] NULL値の確認

### 4. 過去の作業履歴の確認
- [ ] todo.md から過去のデータベース構築セッションを確認
- [ ] チェックポイントのコミットメッセージを確認
- [ ] データ投入スクリプトの実行履歴を確認

## 報告前の最終確認

### ユーザーへの報告前に必ず確認すること
1. ✅ データベースに実データが存在するか？
2. ✅ データ件数は妥当か？（0件ではないか？）
3. ✅ 主要な都道府県（神奈川県、東京都など）のデータは存在するか？
4. ✅ 査定APIが実際に動作するか？

### 報告時の注意事項
- **推測で報告しない**: 必ず実データを確認してから報告する
- **todo.mdだけを信じない**: 実際のデータベース状態を確認する
- **「未完了」と報告する前に**: 実データが存在しないか徹底的に確認する

## このチェックリストの使用タイミング
- ユーザーへの進捗報告前
- データベース関連の質問を受けた時
- 新しいセッション開始時
- チェックポイント作成前

## 再発防止の誓約
データベースの実態を確認せずに推測で報告することは、ユーザーの信頼を損なう重大な過失である。
このチェックリストを必ず実施し、正確な情報のみを報告する。
